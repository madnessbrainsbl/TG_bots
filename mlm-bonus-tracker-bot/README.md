# MLM Bonus Tracker Bot (Telegram)

Телеграм-бот для партнёрской (MLM) сети: регистрация партнёров и лидов, заявки на сделки, автоматические начисления по 10 линиям, статусы партнёров, личный кабинет, выплаты, новости/обучение, админ-панель и уведомления.

## 🔑 Основные функции

1. **Регистрация партнёров**
   - Дерево до 10 линий
   - Без дублей по телефону и Telegram ID
   - ФИО, телефон
   - Выдача реф-кода, реф-ссылки, QR-кода

2. **Добавление клиентов (лидов)** партнёрами
   - Без дублей по телефону
   - Автопривязка партнера по реф-ссылке
   - ФИО клиента, телефон, город, сумма долга, комментарий

3. **Заявки на сделки** и подтверждение админом
   - Создание заявок партнёрами
   - Подтверждение/отклонение админом

4. **MLM начисления**
   - Автоначисления по 10 линиям: L1=1000 … L10=100
   - Прогресс-бар и детализация бонусов по уровням

5. **Статусы партнёров**
   - Автоматическое повышение по лестнице: Бронза → Серебро → … → Элитный
   - На основе подтверждённых сделок и прайс-планов

6. **Личный кабинет**
   - Баланс (потенциал/подтверждено)
   - Заявка на вывод средств
   - Сохранение реквизитов для выплат
   - Статистика: лиды, консультации, договоры

7. **Контент (новости/обучение)**
   - Учёт просмотров, реакции 👍/👎
   - Базовая статистика
   - Роль контент-менеджера
   - Уведомления о непрочитанном контенте

8. **Отзывы партнёров**
   - Модерация админом
   - Партнёры могут предлагать отзывы и новости

9. **Админ-панель**
   - Подтверждение сделок/бонусов
   - Списки партнёров/лидов/выплат
   - Фильтры и экспорт CSV
   - Навигация с кнопками «Назад»/«Вперёд»

10. **Уведомления** (автоматические)
    - Новый партнёр по сети (всем без персональных данных)
    - Новый лид (только админу и модератору)
    - Подтверждённая сделка (по сети без персональных данных)
    - Смена статуса (по сети без персональных данных)
    - Новая заявка на выплату (только админу)

11. **Команды и навигация**
    - /start, /help, /home, /admin
    - Рабочие ReplyKeyboard
    - Ссылки на соцсети (Instagram, Telegram-канал, чат с юристами)

12. **Согласие на ПДн при старте**
    - Обязательное согласие с возможностью ссылки на документ
    - Поддержка HTML-ссылок

**КРИТИЧЕСКИ ВАЖНО**: Перед первым запуском нужно создать таблицы в базе данных.

```bash

```


## 🔧 Решение проблем

### ❌ Ошибка: `sqlite3.OperationalError: no such table: users`

**Причина**: Таблицы в базе данных не созданы.

**Решение**:
1. Убедитесь, что выполнили инициализацию БД (см. пункт 3 выше)
2. Проверьте, что все URL в `.env` одинаковые:
   ```env
   DB_URL=sqlite+aiosqlite:///./bot.db
   DATABASE_URL=sqlite+aiosqlite:///./bot.db
   ```
3. Выполните миграции:
   ```bash
   alembic upgrade head
   ```

### ❌ Проблема: Несогласованность Base

В проекте есть два определения `Base`:
- `db/models.py`: `Base = DeclarativeBase`
- `db/db.py`: `Base = declarative_base()`

**Важно**: Используйте ОДИН `Base` везде. Рекомендуется импортировать `Base` из `db.models`:

```python
# db/db.py
from db.models import Base  # вместо создания нового Base
```

### ❌ Проблема: Разные пути к БД

Alembic и runtime должны использовать одну и ту же базу данных:
- Установите одинаковые `DB_URL` и `DATABASE_URL` в `.env`
- Проверьте, что `alembic.ini` использует правильный URL

### 🔍 Проверка состояния

```bash

# Посмотреть структуру таблицы users

# Проверить статус миграций
```
